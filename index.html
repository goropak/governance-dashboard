    외출 모드 — 원격 자동 협의 + 텔레그램 결재

    외출 중 보좌관(Opus)과 시장(Sonnet)이 메일박스 파일로 자동 협의하고, 핵심 결정(git push)은 핸드폰 텔레그램으로 ✅/❌ 결재하는 흐름.

    구성 요소

    • 보좌관 = assistant-workspace/ — 판단·패치 작성
    • 시장 = governance/ — 실제 집행
    • 메일박스 = governance/active/outbox-to-mayor.md (보좌관→시장), inbox-from-mayor.md (시장→보좌관)
    • 오케스트레이터 = orchestrator/orchestrator.py — 두 파일을 10초마다 감시, 변경되면 상대편을 자동으로 깨움
    • 게이트 = telegram-gate/gate.py — push-pending.md 감시 → 텔레그램 ✅/❌ → 승인 시 자동 git push

    최초 1회 준비

    cd ~/Desktop/project/telegram-gate
pip install -r requirements.txt
# .env 확인 (봇 토큰·chat_id·GOVERNANCE_PATH 이미 설정됨)

    외출 전 실행 순서

    1
    첫 지시 적기 — governance/active/outbox-to-mayor.md에 시장에게 줄 첫 지시를 기록. 이 파일에 내용이 있어야 루프가 시작됨.
    2
    터미널 A — 게이트(결재 창구) 켜기 
    cd ~/Desktop/project/telegram-gate
python3 gate.py
    3
    터미널 B — 오케스트레이터(자동 협의 루프) 켜기 
    cd ~/Desktop/project/orchestrator
python3 orchestrator.py
    4
    외출 — 보좌관↔시장이 outbox/inbox를 주고받으며 자동 협의. push가 필요한 지점에서 시장이 push-pending.md에 신호를 남김.
    5
    핸드폰 결재 — 텔레그램에 “📤 push 승인 요청” + 요약 + ✅승인/❌거부 도착. ✅ 누르면 게이트가 본인 ID 검증 후 자동 git push → “✅ 승인 + push 완료” 회신.
    6
    정지 
    touch ~/Desktop/project/STOP   # 즉시 정지 (또는 각 터미널 Ctrl+C)

    안전장치

    • STOP 파일 감지 시 즉시 정지.
    • push 없이 20사이클 돌면 자동 정지(사람 확인 필요).
    • 보좌관이 할 일 없으면 outbox에 “대기”만 적어 정상 종료.
    • 깨우기 간 최소 20초 간격, 호출당 최대 10분 타임아웃.
    • 게이트: 본인 텔레그램 ID 검증, 동일 신호 재알림 방지(SHA), 커밋 해시 앞 7자만 노출.

    요약: 게이트 켜기 → outbox에 첫 지시 → 오케스트레이터 켜기 → 외출 → 핸드폰 ✅/❌
