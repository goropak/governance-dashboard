# 대통령 프로필 Vault — 사용설명서

> 이 문서는 vault의 **목적·구조·사용법**만 설명한다(시스템 설명). 개인 내용(성격·재무 등)은 담지 않으므로 외부 노출에 안전하다. 실제 내용은 별도 private 레포 `president/`에 있다. (포인터: `briefing/president-vault.md`)

## 1. 목적 — 왜 만드나

원초적 통증은 "여러 프로젝트와 결정을 굴리다 보면, 정작 *나 자신*(성향·우선순위·제약)을 매번 다시 설명해야 한다"는 것. 이 vault는 그걸 한 곳에 모은 **"대통령 운영 매뉴얼"**이다.

성숙하면:
1. **자동 필터** — 새 도시 착수·큰 결정 신호 때 후보를 본인 기준(장기·무인화·비동기·품질)에 비춰 거른다.
2. **레드/그린 플래그 게이트** — "계속 붙어 있어야 하는 구조"면 착수 전 경고.
3. **선제 경고 거울** — 번아웃·반복 실수 패턴을 쌓아 재현 조짐을 미리 짚는다.

도시(프로젝트)가 **아니다** — 코드·인프라·시장 작업 없음. 거버넌스의 **참조 레이어**다.

## 2. 무엇이 들어있나 (7 카테고리)

| 파일 | 내용 | 태그 |

| 02 성격·기질 | 일하는 방식·강약점·정보소비 스타일 | `#성격` |

| 05 목표·위시리스트 | 단기·장기 목표, 프로젝트 | `#목표` |

## 3. 언제 쓰나 (보좌관)

새 도시 착수·중요 의사결정 신호가 보이면 참조한다. **부팅 자동 로드는 안 한다**(경량 부팅 원칙). 필요할 때 태그로 당긴다:

```
grep -rl "#의사결정" president/    # 결정 스타일
grep -rl "#목표"     president/    # 무엇을 하고 싶은지
grep -rl "#제약"     president/    # 현실성 체크(시간·기술)
```

## 4. 어떻게 추가/수정하나 (대통령)

- 편집 도구: **Obsidian**(데스크톱·Android 모두). vault는 그냥 마크다운 + `[[위키링크]]`라 보좌관도 같은 파일을 읽는다.
- 빈칸은 `(미작성)`으로 두고, 떠오를 때 채운다. 모바일에서 한 줄씩 적어도 됨.
- 확실치 않은 추론은 `(추정·확인 필요)`로 표기 — 사실과 구분.
- 변하는 사실(현재 우선순위 등)은 **날짜**를 함께 적는다.
- 1사실 1원본(제7조) — 같은 사실을 두 파일에 적지 말고 가리킨다.
